# 实现文档收束归纳

## 1. 能力目标
- 在 Openclaw 中接入图片与视频识别能力。
- 当当前对话模型不支持视觉输入时，使用支持视觉的模型完成识别，再将识别结果返回主流程。

## 2. 模型能力边界（文档给定）
- `qwen3.5-plus`、`kimi-k2.5`：支持视觉输入。
- 其他列举模型（如 `qwen3-coder-*`、`glm-*`、`MiniMax-M2.5`）：文档标注为不直接支持视觉输入，需要通过视觉技能链路补齐。

## 3. 输入与路由策略
- 图片输入：支持公网 URL 与本地文件。
- 视频输入：支持公网 URL 与本地文件。
- 统一路由原则：
  - 本地小文件走 Base64 直传。
  - 本地超限视频走抽帧后多图输入。
  - 可用公网地址时优先按 `image_url` / `video_url` 结构传入。

## 4. 图片处理约束（文档给定）
- 默认文件大小限制：`<= 7MB`。
- 默认分辨率限制：`<= 4K`（最大边 `<= 4096`）。
- 格式范围：
  - 4K 以下：`BMP/JPEG/PNG/TIFF/WEBP/HEIC`。
  - 4K-8K：`JPEG/PNG`。
- 命中限制时返回限制项与触发条件。

## 5. 视频处理约束（文档给定）
- `video_url` 公网模式：
  - `qwen3.5/Qwen3-VL` 最高 `2GB`（依赖远端返回 `Content-Length` 与 `Content-Type`）。
- Base64 模式：
  - 总请求体需小于 `10MB`。
  - 本地文件按 `<= 7MB` 作为直传阈值。
- 本地超限视频处理：
  - 抽帧后按多图请求发送。
  - 帧数上限 `<= 8000`。
  - 文档中给出两种抽帧频率描述：`2 fps` 与 `10 fps`。

## 6. 请求结构规范
- 单图：`content` 中包含一个 `image_url` 与文本指令。
- 多图对比：`content` 中放置多个 `image_url` 与文本指令。
- 视频直传：`content` 中使用 `video_url` 与文本指令。
- 视频抽帧：`content` 中放置多张帧图 `image_url`（可为 Data URL）与文本指令。

## 7. 并发与接口一致性
- 批量识别与对比场景使用异步调用（`AsyncOpenAI`）。
- API Key 与 Base URL 与 Openclaw 现有配置保持一致，不引入新的环境变量约束。

## 8. 返回行为
- 返回模型原始分析结果。
- 返回输入处理路径（URL 直传、Base64、抽帧）。
- 返回命中限制信息（大小、分辨率、格式、帧数等）。
